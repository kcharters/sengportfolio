So that only the person who had created the game or score could edit and no one else. We hadnt impemented any foreign keys at this statge so i added column migrations to add that to our already exisiting migrations for games and scores php artisan make:migration add_foreign_key_column_to_games_table --table=games (and the same but for scores) After adding a foreign key I had to make a relationship in the model between the user and the games.

    then I used the user id foreign key to add to each creation of a game and score
